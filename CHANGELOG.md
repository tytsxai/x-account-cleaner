## [0.1.0] - 2026-05-19

首个 tag 化的版本,codify 当前 main 的工具能力。

### 包含

- **六种清理任务**(各自独立可配置开关):
  - 批量删除全部推文
  - 批量删除全部回复
  - 批量取消全部转推
  - 批量取消全部点赞
  - 批量删除全部书签
  - 批量取消关注
- **登录管理**:自动 + 手动登录,会话持久化(Playwright user-data 目录)
- **配置驱动**:启用/禁用单项任务、调整批次大小与节流间隔
- **指数退避重试**:瞬时失败自动重试
- **完整日志**:文件审计轨迹
- **反检测**:模拟真实用户浏览器指纹与操作节奏
- **跨平台脚本**:Windows `install.bat` + `start.bat`,macOS/Linux `install.sh`
- **实时进度**:按类别 + 总体计数
- **发现性**:双语关键词块 + `llms.txt`

### 安全提醒

- **操作不可恢复** —— X 没有撤销
- **ToS 责任在用户** —— 工具以你自己的登录身份运行,但批量操作可能受 X 政策限制
- **可能触发限流 / 软封** —— 已有分批与节流但不能完全规避;建议先用小号试
- **不触碰**:DMs、列表、社群、付费功能

## [1.2.0] - 2025-10-05

### 🎯 重大更新：独立选择器配置系统

#### 新增

- ✨ **独立选择器配置文件** `selectors.json` - 应对 Twitter 页面结构更新
- ✨ **主选择器 + 备用选择器** - 提高选择器稳定性
- ✨ **选择器热更新** - 无需修改代码，只需更新配置文件
- 📝 **详细更新指南** `docs/SELECTOR_UPDATE_GUIDE.md` - 完整的教程和示例
- 📝 **快速参考文档** `选择器配置说明.md` - 快速上手指南
- 💾 **配置备份文件** `selectors.backup.json` - 防止配置丢失
- 🔧 **扩展类型系统** - 新增 `SelectorConfig` 和 `SelectorsJsonConfig`

#### 改进

- 🔧 **配置加载逻辑** - 支持 `selectors.json` > `config.json` 优先级
- 🔧 **SelectorHelper** - 支持动态选择器配置
- 📖 **README 文档** - 新增选择器配置说明章节
- 💡 **用户体验** - 启动时显示选择器配置版本信息

#### 核心价值

**为什么需要这次更新？**
Twitter 经常更新页面结构，导致元素选择器失效。通过独立的配置文件，用户可以：
- ✅ 快速应对页面结构变化
- ✅ 无需等待程序更新
- ✅ 社区可以共享最新配置
- ✅ 降低维护成本

**如何使用？**
1. 当功能失效时，打开 `selectors.json`
2. 使用浏览器开发者工具（F12）查找新选择器
3. 更新配置文件中的对应项
4. 重新运行程序即可

详细教程：[选择器配置说明.md](选择器配置说明.md)
