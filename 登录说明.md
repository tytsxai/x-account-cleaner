# 登录说明

本工具支持两种登录方式：手动登录（默认）与自动登录（可选）。

## 方式一：手动登录（推荐）

1. 不需要配置 `.env`
2. 启动后会打开浏览器并进入登录页
3. 在浏览器内完成登录与可能的安全验证
4. 程序检测到已认证页面 DOM 后自动继续执行

手动登录等待会识别登录表单、验证码/二次验证、账号访问限制和页面延迟渲染。进入验证步骤时会额外延长一次等待时间，完成验证后不要求 URL 必须停在 `/home`。

## 方式二：自动登录（可选）

1. 复制 `env.example` 为 `.env`
2. 填写账号信息：
   - `TWITTER_USERNAME`
   - `TWITTER_PASSWORD`
3. 运行程序后将自动登录

自动登录会按阶段重试：进入登录页、输入用户名、检测验证 detour、输入密码、提交、等待已认证 DOM。若出现验证码、2FA、账号访问限制或身份确认：

- `HEADLESS=false`：保留当前页面并切换到手动等待
- `HEADLESS=true`：明确失败，需要改为有头模式人工完成验证

## 登录状态保存

登录主会话会保存在 `browser-data/profile/`，并额外写出 `browser-data/state.json` 快照。若只想重新生成快照，删除 `state.json` 后运行即可；若需要完整重新登录，请同时删除 `browser-data/profile/` 后以有头模式运行。

## 安全提示

- `.env` 包含敏感信息，请勿分享或提交到仓库
- 若开启了 2FA，仍可能需要手动输入验证码
- 使用 `HEADLESS=true` 时，需提供账号密码或已保存登录状态
- URL 进入 `/home` 不等于登录成功，程序会等待账号菜单、个人资料导航等已认证 DOM 信号
